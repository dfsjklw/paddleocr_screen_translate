# screen_transalate_ocr — PaddleOCRv5 ncnn 命令行 OCR 工具

基于 [PaddleOCRv5_ncnn](https://github.com/zhahoi/PaddleOCRv5_ncnn) 的 Windows 命令行 OCR 识别工具，输出带位置坐标的 JSON 结果。

## 使用说明

```
screen_transalate_ocr.exe <mode> <path> [options]
```

### 参数

| 参数 | 说明 |
|---|---|
| `single` | 处理单张图片，输出 `output/output.json` |
| `folder` | 处理文件夹内所有图片，每张图片输出独立的 `output/<文件名>.json` |
| `path` | 图片文件路径（single 模式）或文件夹路径（folder 模式） |
| `--draw` | **可选**。启用后同时保存标注了识别结果的图片（`output/result.jpg`）并弹窗显示 |
| `--char-thresh FLOAT` | **可选**。单字符最低置信度（0.0~1.0，默认 0.0 = 关闭过滤）。低于此值的字符会被丢弃 |
| `--block-thresh FLOAT` | **可选**。文本块平均置信度阈值（0.0~1.0，默认 0.0 = 关闭过滤）。平均置信度低于此值的整个文本块会被丢弃 |
| `--box-thresh FLOAT` | **可选**。检测框轮廓分数阈值（0.0~1.0，默认 0.6）。降低可检测到更多文本区域，提高则更严格 |
| `--binary-thresh FLOAT` | **可选**。概率图二值化阈值（0.0~1.0，默认 0.3）。降低可检测到更多候选区域，提高则更严格 |
| `--dict FILE` | **可选**。字典文件路径（默认 `weights/zh_dict.txt`，可用 `weights/en_dict.txt` 识别英文） |

### 检测灵敏度调节

| 方向 | `--box-thresh` | `--binary-thresh` | `--char-thresh` | `--block-thresh` |
|---|---|---|---|---|
| **更宽松**（检测更多文本） | 降低（如 0.4） | 降低（如 0.15） | 保持 0.0 | 保持 0.0 |
| **更严格**（减少误识别） | 提高（如 0.7） | 提高（如 0.4） | 提高（如 0.3） | 提高（如 0.4） |

### 使用示例

```batch
REM 单张图片识别，仅输出 JSON（默认参数）
screen_transalate_ocr.exe single C:\images\screenshot.png

REM 单张图片识别，输出 JSON + 标注图片
screen_transalate_ocr.exe single C:\images\screenshot.png --draw

REM 更宽松的检测 —— 捕获更多弱对比度/模糊文字
screen_transalate_ocr.exe single C:\images\screenshot.png --box-thresh 0.4 --binary-thresh 0.15

REM 更严格的检测 —— 减少误识别
screen_transalate_ocr.exe single C:\images\screenshot.png --box-thresh 0.7 --char-thresh 0.3 --block-thresh 0.4

REM 英文识别（使用英文词典）
screen_transalate_ocr.exe single C:\images\screenshot.png --dict weights/en_dict.txt

REM 批量处理文件夹
screen_transalate_ocr.exe folder C:\images\

REM 批量处理文件夹 + 标注图片
screen_transalate_ocr.exe folder C:\images\ --draw
```

### 输出格式 (`output.json`)

```json
{
    "input_path": "screenshot.png",
    "page_index": null,
    "model_settings": {
        "use_doc_preprocessor": false,
        "use_textline_orientation": false
    },
    "doc_preprocessor_res": {
        "input_path": null,
        "page_index": null,
        "model_settings": {
            "use_doc_orientation_classify": false,
            "use_doc_unwarping": false
        },
        "angle": -1
    },
    "dt_polys": [[[x0,y0],[x1,y1],[x2,y2],[x3,y3]], ...],
    "text_det_params": {
        "limit_side_len": 64,
        "limit_type": "min",
        "thresh": 0.3,
        "max_side_limit": 4000,
        "box_thresh": 0.6,
        "unclip_ratio": 1.5
    },
    "text_type": "general",
    "textline_orientation_angles": [-1, ...],
    "text_rec_score_thresh": 0.0,
    "text_rec_params": {
        "char_confidence_thresh": 0.0,
        "block_confidence_thresh": 0.0
    },
    "return_word_box": true,
    "rec_texts": ["text1", "text2", ...],
    "rec_scores": [0.99, 0.87, ...],
    "rec_char_avg_scores": [0.99, 0.95, ...],
    "rec_polys": [[[x0,y0],[x1,y1],[x2,y2],[x3,y3]], ...]
}
```

### 字段说明

| 字段 | 说明 |
|---|---|
| `input_path` | 输入图片路径 |
| `dt_polys` | 每个文本块的四个角点坐标 `[[x,y], ...]`（逆时针） |
| `text_det_params` | 运行时实际使用的检测参数（随 `--box-thresh`/`--binary-thresh` 变化） |
| `text_rec_params` | 运行时实际使用的识别过滤参数（随 `--char-thresh`/`--block-thresh` 变化） |
| `rec_texts` | 每个文本块的识别文字 |
| `rec_scores` | 每个文本块的检测轮廓置信度（0~1） |
| `rec_char_avg_scores` | 每个文本块的平均字符识别置信度（0~1），反映文字识别质量 |
| `rec_polys` | 与 `dt_polys` 相同，每个文本块的位置坐标 |
| `textline_orientation_angles` | 文本方向：`-1`=水平，其他值=垂直 |

## 依赖说明

### 运行时依赖

| 依赖 | 版本 | 说明 |
|---|---|---|
| [ncnn](https://github.com/Tencent/ncnn) | 20260526 | 腾讯高性能神经网络推理框架（VS2022 预编译） |
| [opencv-mobile](https://github.com/nihui/opencv-mobile) | 4.13.0 | 轻量级 OpenCV（VS2022 预编译，静态库） |
| PaddleOCRv5 模型权重 | — | `weights/` 目录下的 `.param`/`.bin` 文件和字典文件 |

### 目录结构要求

```
screen_transalate_ocr.exe
├── weights/
│   ├── PP_OCRv5_mobile_det.ncnn.param
│   ├── PP_OCRv5_mobile_det.ncnn.bin
│   ├── PP_OCRv5_mobile_rec.ncnn.param
│   ├── PP_OCRv5_mobile_rec.ncnn.bin
│   ├── zh_dict.txt
│   └── en_dict.txt
└── output/          （自动创建）
    ├── output.json  （single 模式输出）
    └── result.jpg   （仅 --draw 模式）
```

### 编译依赖（仅开发需要）

| 依赖 | 说明 |
|---|---|
| Visual Studio 2026 (v145) | 编译器和工具链 |
| ncnn-20260526-windows-vs2022 | 头文件和静态库 |
| opencv-mobile-4.13.0-windows-vs2022 | 头文件和静态库 |

**编译配置要点：**
- 平台工具集：`v145`
- C++ 标准：`C++20`
- 运行库：`/MD`（Release）/ `/MDd`（Debug）
- 预处理器：`NOMINMAX`（避免 windows.h 的 min/max 宏冲突）、`_CRT_SECURE_NO_WARNINGS`
- 编译器选项：`/utf-8`、`/openmp`、`/bigobj`
- 禁用 `/GL`（全程序优化），因链接的库为 VS2022 编译
